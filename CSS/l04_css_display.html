display, visibility, opacity 속성으로 노드의 레이아웃 제어

display: block

항상 새로운 라인으로 시작하고 가로폭 전체를 차지하려는 특징을 가진다.
width의 default : 100%

  • width가 지정될 시 margin이 남은 가로폭(margin-right)을 채운다.
  • margin-left의 default: 0
  • margin-right의 default: auto
  • 보통 block은 inline 요소를 포함하기 위해 지정한다.
  • box model의 속성을 모두 포함한다.
    • width: 넓이, max-width: 최대넓이, min-width: 최소넓이
    • height: 높이, max-height: 최대높이, min-height: 최소높이
    • padding: content와 border 사이 여백
    • border: box의 윤곽선( [length | percentage | auto] | style | color )
    • margin: box의 외부 여백
      • [padding | margin | border의두께] 표기
      • padding: top | right | bottom | left
      • padding: top | horizontal | bottom
      • padding: vertical | horizontal
      • padding: all four sides

display: inline

test

줄을 바꾸지 않고 다른 요소와 한 행에 위치한다.

  • width와 height, margin-top, margin-bottom 값을 지정할 수 없다.
  • width,height의 값을 지정할 수 없지만 가로 세로 길이는 fit-content와 동일하게 작동한다.
  • 하지만 line-height로 상하 여백을 지정할 수 있다.
  • inline의 스타일을 변경하거나 쓰임을 명시하기 위해 작성한다.

display: inline-block

테스트 test

inline과 같이 줄을 바꾸지 않고 다른요소와 한 행에 위치하지만, block처럼 box model의 속성 모두를 지정할 수 있다.

  • 요소 내부 컨텐츠의 위쪽여백을 4px로 자동 지정한다.
  • width의 default: fit-content
  • 같은 level의 요소의 content와 inline-box안의 content가 세로정렬이 같이 된다.
  • vertical-align 으로 컨텐츠의 세로 정렬이 가능하다.

display: none

요소를 화면에서 지운다.

  • 해당 요소를 화면에서 지운다.
  • 해당 요소의 노드 개체가 삭제되지는 않는다.

visibility

요소의 가시 유무를 결정한다.

  • visibility: hidden은 해당 요소를 화면에서 보이지 않게 한다.
  • display: none과 다르게 화면에서 요소를 지우지 않는다.
  • table의 열이나 행이 사라지지 않을 때는 collapse나 none을 사용해야한다.
  • visibility: visible은 default 값으로 해당 요소를 보이게한다.
  • visibility는 애니메이션 대상에서 포함되어 있지만 오작동한다.

opacity

요소의 투명도를 설정한다.

  • opacity: 0은 visibility: hidden과 거의 동일하게 동작한다.
  • 요소의 투명도는 [0,1]로 조정이 가능하다.
  • rgba는 배경, 텍스트 등의 색상,투명도를 변경하는 것으로, 요소 자체의 투명도를 조절하는 opacity와는 차이가 있다.

transform scale(x,y)

요소의 투명도를 설정한다.


  • style.transform 은 노드 개체를 2D 또는 3D로 변환해서 회전, 배율, 이동, 기울기 등을 변경한다.
  • transform을 적용한 노드는 실제 크기가 변경되는 것이 아니라 스냅샷 같은 것이 변경되어서 화면에 표현되기 때문에 다른 개체를 밀어내거나 당기지 않는다.
  • transform은 애니메이션을 적용하기에 최적의 속성이다.
