자바스크립트의 변수(variable)

모든 프로그래밍 언어는 데이터를 참조하기 위해 변수를 사용한다. 이 때, 참조하는 데이터가 변할 수 있어서 변수다.
ex)let a="안녕"; a=200;

자바스크립트의 변수와 상수

  • var : 자바스크립트가 생성될 때부터 사용되었던 변수선언 예약어(keyword)로 함수지향언어의 특징을 가진 block이 모호한 상태다. ( 앞으로 사용하지 않음!!! )
  • let : es6에 탄생한 변수로 클래스 추가와 함께 block의 구분을 명확하게 한다. (var 대신 사용)
  • const : 변수가 아닌 상수로 선언과 동시에 데이터를 참조해야 하고, 참조된 데이터를 변경할 수 없다.
    ( 참조하는 데이터의 상태를 변경할 수 있다. )

변수의 식별자(이름)

프로그래밍에서 식별자, 즉 이름 짓기는 매우 중요합니다. 대체로 유지보수에 용의하기 위해 약속된 방식이지만, 이름이 오류를 만들 수 있어 오류가 있는 코드를 배포하는 자바스크립트 특성상 이름 짓기가 더 중요하다.
ex) let var=10; or let function="test"; ( error, 예약어를 식별자로 사용 )

  • 변수 이름은 문자, 숫자, 밑줄('-' 제외), $ 가 포함될 수 있다.
  • 변수는 숫자로 시작하는 것을 권장하지 않는다. ex) let 1num; (x) elt num1; (o)
  • 변수의 이름은 구체적이어야 하고 문자와 문자의 결합을 카멜 표기법(camel) or 파스칼 표기법(pascal), 스네이크 표기법(snake)로 구분한다.
    카멜 표기법 : javaScript, userId
    파스칼 표기법 : JavaScript, UserId
    스네이크 표기법 : member_num, user_id
  • 상수(const)의 이름은 모두 대문자이고, 스네이크 표기법으로 표기한다.
    ex) COLOR_RED
  • 이름은 대소문자를 구분한다.
    ( num, Num, NUM : 모두 다른 변수이다.)
  • 예약어는 변수 이름으로 사용할 수 없다.
    ex) function, var, let, Number, ...

그 밖의 규칙(권장)

  • 읽고 이해할 수 있는 이름을 사용
    ex) userName, colorName, colorHexCode
  • a,b,c,i 같은 짧은 이름은 피한다. 단, 구조적으로 변수명이 명확할 때 사용가능
    ex) for(let i=0; i<len; i++) : i는 index
  • 소속된 팀의 규칙을 따른다.
    ex) userId와 memberNum를 혼용해서 사용 (x)
